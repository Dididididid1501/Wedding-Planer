 Ядро приложения (Core)
Файлы: js/core/storage.js, js/core/events.js, js/core/utils.js

StateManager (хранилище состояния)
Централизованно хранит все данные приложения: задачи, категории, расходы, гостей, столы, состояние свёрнутости секций.

Автоматически загружает данные из localStorage при старте и сохраняет при любых изменениях.

Предоставляет демо-данные при первом запуске.

Методы getState(), setState(updates), subscribe(listener) позволяют модулям получать и обновлять данные, а также реагировать на изменения.

EventBus (шина событий)
Обеспечивает слабую связь между модулями через события.

Методы on(event, callback), emit(event, data), off(event, callback).

Примеры событий: state:update (обновление данных), stateChanged (уведомление после сохранения), guests:updated (для синхронизации рассадки).

Утилиты
Набор вспомогательных функций: генерация уникальных ID, экранирование HTML, форматирование чисел и дат, capitalize, parseNumber и др.

📋 Модуль «План подготовки» (Tasks)
Файлы: js/modules/tasks/*

Возможности для пользователя
Категории задач
Можно создавать свои категории (например, «Декор», «Подрядчики») и выбирать для них цвет из палитры 20 оттенков. Категории используются для группировки задач и отображаются цветными бейджами.

Задачи

Поля: название, категория, ответственный (Невеста, Жених, Семья, Агентство, Другое), месяц начала и дедлайн, статус, комментарий.

Статус может быть «Запланировано», «В работе», «Выполнено» — влияет на прогресс и внешний вид (зачёркивание).

Для каждой задачи можно отмечать выполненные месяцы через чекбоксы (промежуточные этапы). При отметке всех месяцев задача автоматически становится выполненной, при частичной — «в работе».

Прогресс-бар
Визуально показывает долю выполненных этапов от общего количества месяцев по всем задачам. Разделен на три зоны: выполнено (зелёный), частично (оранжевый), запланировано (серый).

Фильтрация
Можно отфильтровать задачи по категории, месяцу (учитывается месяц начала, дедлайна или отметка выполнения) и статусу.

Управление
Добавление, редактирование (в модальном окне) и удаление задач и категорий.

Технические детали
Модель (TasksModel) содержит логику работы с задачами и категориями, автоматический пересчёт статуса, фильтрацию и подсчёт прогресса.

Представление (TasksView) генерирует HTML-формы, таблицу, прогресс-бар, модальные окна и привязывает события.

Контроллер (TasksController) связывает модель и представление, обрабатывает действия пользователя и эмитит события state:update при изменениях.

💰 Модуль «Бюджет» (Budget)
Файлы: js/modules/budget/*

Возможности для пользователя
Статьи расходов

Поля: название, планируемая сумма, ответственный (Невеста, Жених, Организатор), заметка.

Каждая статья может иметь несколько платежей.

Платежи

У каждого платежа есть дата, сумма и чекбокс «оплачено».

Можно добавлять и удалять платежи как прямо в строке таблицы, так и в модальном окне редактирования.

Суммы оплаченных платежей автоматически суммируются.

Автоматические расчёты

Для каждой статьи показывается оплаченная сумма и остаток.

В футере таблицы отображается общая сумма бюджета, всего оплачено и остаток по всем статьям.

Технические детали
Модель (BudgetModel) управляет массивом расходов и методами для добавления/обновления/удаления статей и платежей, а также подсчёта итогов.

Представление (BudgetView) рендерит таблицу с динамическими полями ввода (дата, сумма, чекбокс) и обрабатывает изменения через делегирование событий.

Контроллер синхронизирует изменения с глобальным состоянием через state:update.

👥 Модуль «Список гостей» (Guests)
Файлы: js/modules/guests/*

Возможности для пользователя
Анкета гостя
Обширный набор полей, разбитый на три блока:

Основное: имя, отношение (Семья жениха/невесты/Другое), приглашён, будет ли в ЗАГСе, приведёт ли с собой гостей, номер стола, группа конфликта.

Контакты: email, адрес, потребность в размещении, транспорт (не нужен / нужен туда / обратно / туда-обратно).

Предпочтения: тип питания (стандарт, веган, детское, безглютен), выбор блюда, отметки об алкоголе (шампанское, красное/белое вино, крепкое, непьющий), заметка.

Приведённые гости
При выборе «Приведёт: Да» открывается блок, где можно добавить одного или нескольких сопровождающих. Для каждого указывается пол (муж/жен/ребёнок) и опция «детский стол» (для детей). Эти гости автоматически создаются как отдельные записи с пометкой «приведён».

Фильтрация и статистика

Поиск по имени и фильтр по отношению.

Статистические карточки: общее количество, приглашённые, распределение по отношениям, потребление алкоголя.

Редактирование
Любого гостя можно отредактировать в модальном окне (все поля доступны для изменения).

Технические детали
Модель (GuestsModel) содержит методы для добавления (с учётом приведённых), обновления, удаления и фильтрации гостей, а также сбора статистики.

Представление (GuestsView) отрисовывает сложную форму добавления, таблицу, панель статистики и управляет динамическим списком приведённых гостей.

Контроллер обрабатывает действия, вызывает модель и эмитит state:update и guests:updated (для обновления рассадки).

🪑 Модуль «План рассадки» (Seating)
Файлы: js/modules/seating/*

Возможности для пользователя
Визуальное управление столами
Каждый стол представлен карточкой, в которой отображаются гости, сидящие за ним.

Карточка показывает вместимость (можно менять числом), количество занятых и свободных мест.

При переполнении стола рамка становится красной.

Если за столом есть гости с одинаковой группой конфликта, появляется предупреждение.

Drag-and-drop рассадка
Гостей можно перетаскивать между столами или «выбросить» на пустое пространство, чтобы убрать со стола. Также есть кнопка «Пересадить» для выбора стола из выпадающего списка.

Управление столами

Добавление нового стола (автоматический следующий номер).

Настройка вместимости через шестерёнку (модальное окно) или прямо в карточке.

Кнопка очистки стола (убирает всех гостей).

Автоматическая рассадка
Одним кликом можно равномерно распределить всех нерассаженных гостей по существующим столам.

Статистика
В верхней части секции отображается общее количество мест, сколько гостей уже сидят за столами и сколько свободных мест.

Технические детали
Модель (SeatingModel) работает со списком столов и гостей: добавляет/обновляет столы, сажает гостей, очищает столы, проверяет конфликты, синхронизирует столы с номерами из анкет гостей.

Представление (SeatingView) строит сетку карточек столов с обработкой drag-and-drop, отображает гостей и управляет модальными окнами настроек и пересадки.

Контроллер подписан на событие guests:updated, чтобы актуализировать столы при изменении гостей, и эмитит state:update при любых изменениях рассадки или столов.

🧩 Интеграция и общие функции
Сворачивание секций
Каждая из четырёх секций (Задачи, Бюджет, Гости, Рассадка) может быть свёрнута/развёрнута кликом по заголовку. Состояние сохраняется в localStorage.

Сохранение данных
Любое действие (добавление, редактирование, удаление, перетаскивание, отметка чекбокса) немедленно сохраняет всё состояние в localStorage. При обновлении страницы данные восстанавливаются.

Демо-данные
При первом запуске приложение автоматически наполняется демонстрационными задачами, расходами, гостями и столами, чтобы сразу показать возможности.

Адаптивный дизайн
Интерфейс оптимизирован для экранов от 640px, таблицы имеют горизонтальную прокрутку на узких экранах. Предусмотрен минимальный стиль для печати (скрываются кнопки и лишние элементы).
